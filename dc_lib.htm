DC图书馆应用纲要(DC-Lib)

1.介绍

    在DCMI (Dublin Core Metadata Initiative)中，应用纲要(application profiles，参见 Application profiles: mixing and matching metadata schemas)的概念是作为在特定应用或项目中声明某一元素取自哪个命名域的一种方法而出现的。应用纲要的定义如下：应用纲要是一种取自一个或多个命名域的数据元素组成的元数据格式，由应用者组合并可为了本地的特殊应用进行优化处理。

    DCMI图书馆工作组探索了都柏林核心元数据元素集在图书馆及相关的应用项目中的不同用法，总结出如下可行的用法：

  • 在使用不同元数据标准/格式的系统之间，DC作为交换格式；

  • 在图书馆领域或其他领域，DC作为数据源的收割元数据标准；

  • 在大量不同的系统中，DC支持图书馆对资源的简单编目；

  • 导出MARC数据提供给其他组织机构 (通过转换成 DC)；

  • 使用DC可以从非图书馆的创建者那儿获得用于资源发现的元数据；

    图书馆应用纲要将对以下方面进行详细规范： 

  • 必备的元素
  • 所采用的DC元素
  • 所采用的DC元素修饰词
  • 所采用的编码体系修饰词及值 (举个例子：使用某个特定的受控词表或编码体系)
  • 取自其他命名域、用于图书馆领域的元素
  • 有可能使用的、取自其他应用纲要的元素或元素修饰词 (如DC-Education中的用户对象、即Audience元素)
  • 以标准定义的方式进行限定

    本文档提出了可行的应用纲要，明确了都柏林核心元数据元素集在图书馆或与图书馆相关的应用项目中的用法。它起初由DCMI图书馆工作组下面的DCMI图书馆应用纲要草案委员会提供。

    本次版本在2004年8月份提出，融入了DCMI应用委员会在2003年会议中的决议及2003年9月份在西雅图的工作组会议中所讨论的问题。而且，此文档已经根据由CEN MMI-DC Workshop提出的都柏林核心元数据应用纲要指南(Dublin Core Application Profile Guidelines)做了格式修改，与指南保持一致。

术语类型

术语在语法范畴中的类型。(比如“元素”、“元素限定词”、“编码体系”)。

限定

在语法意义上，被定义的术语是所指引术语的限定。限定词使得元素的意义更为狭义或更为专指。它在一个更加受限的范围内含有未被限定的元素的部分语义。

被限定

在语法意义上，被所指引的术语是对被定义的术语的限定。

编码体系应用于

所定义的术语，也就是某个编码体系，修饰了所指引的术语。使用编码体系有助于理解元素的值的具体含义。这些编码体系包括受控词表及规范的符号(formal notations )或解析规则。用某一编码体系表示的值意味着该值或者是取自某一控制词表的语义符号(如分类体系或主题词表中的术语)，或者是格式符合某种正式编码标准的字符串(如“2000-01-01"作为一个日期的标准表达)。如果一种编码体系无法被客户机或代理器理解，但应用该编码体系的值仍有可能有助于人们对它的理解。

在一些情况下，这里指定的编码体系有可能没有注册。它们将会通过DCMI应用委员会，注册或被提交为“DC编码体系”。

可应用的编码体系

被定义的术语可被所指引的编码体系所修饰。

约束

指明元素是否总是出现或只是有时候出现。在本应用纲要中，“约束”的级别有：必备(M)、有则必备 (MA)、强烈推荐(R) 及可选(O)。必备（M）确保某些元素总被使用，有则必备（MA）表示若能获得有关信息则该元素一定被使用。一个约束级别为必备（M）的元素，肯定有一个值。强烈推荐（R）和可供选择（O）的元素指如果信息适合该资源则应用值填入，如果信息不合适该资源，它们则可省略。

出现次数

指明元素是否可重复及重复的次数。

DC-Lib注释

为明确日期的含义，推荐使用元素限定词。建议如下日期编码形式：
1)使用W3C-DTF (ISO 8601的一个应用，以连字符分隔)
2)使用ISO 8601 (无连字符分隔), 或
3)提供不是从数字字符串中取值的自由文本格式（带或不带连词符）
第二种选择，ISO 8601(无连字符分隔)，是最佳选择。

为了表示公元前的日期、有疑问的日期或近似的日期，将来可能建立起DC-Lib的编码体系或ISO 8601的应用方案。为推进这些问题的解决，日期工作组已经成立。
